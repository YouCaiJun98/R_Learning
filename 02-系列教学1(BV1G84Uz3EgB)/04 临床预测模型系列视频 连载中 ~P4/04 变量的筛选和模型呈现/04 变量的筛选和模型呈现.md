## 变量的筛选
### 为什么需要筛选预测因子
1. 正样本不足
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260408230335324.png)
2. 共线性
    * 有的预测因子之间可能高度相似 / 高度相关：
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260408230514982.png)

### 预测因子筛选的原则
1. 专家意见为主，统计显著性为辅
    * 不能脱离临床。
2. 因果关系为主，关联关系为辅
    * 这条其实不是很重要，因为不太关心具体是因果关系还是关联关系。
    * 在生物学逻辑上认为有因果关系即可，不用非要做因果分析。
3. 原有模型中的预测因子为主，新的预测因子为辅
4. 临床可获得为主，预测效能为辅

### 预测因子的筛选框架
1. 去除缺失较多的预测因子（>20%）
    * 难以获得的预测因子一般就不要了（>15%缺失），特别重要的可以放宽到20%。
    * 还要注意缺失比例比部分变量比例还高的情况。
2. 去除信息较少的预测因子（取值单一）
    * 比如下表中的变量2，它带来的信息量太少了：
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260408231202759.png)
3. 基于单因素分析筛选变量
    * UP主提到，单因素显著和多因素显著没有直接的联系，但是大家都这么做，所以可以考虑跟着这么做🤐
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260408231601122.png)
4. 子集筛选法 - 以最优子集法举例
    * 比如总共有5个变量，共有$2^5=32$种可能（其实就是枚举法🥹）
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260408231820884.png)
5. 正则化技术和其他机器学习技术
    * 还有随机森林、XGBoost、神经网络。
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260408232131785.png)
    * 补充：**正则化**（Regularization）
        * 正则化就是在模型训练时加一个**惩罚项**，防止模型把噪声也学进去，从而**避免过拟合**，让模型更稳定、泛化能力更强。
        * 在原来的损失函数后面，加上对模型系数大小的惩罚：
            $$新损失=原损失+\lambda ⋅ 惩罚项$$
            * $\lambda$越大，惩罚越强，系数会被压得越小
            * $\lambda=0$ 就是普通回归，没有正则化
        * 常用的正则项包括：
            * L1 正则（LASSO），**惩罚系数的绝对值和**，能把不重要变量的系数直接压到 0，实现自动特征选择。
            * L2 正则（Ridge / 岭回归），**惩罚系数的平方和**，让所有系数变小但不为 0，主要解决**多重共线性**，让模型更稳定。
            * Elastic Net（弹性网络），L1 + L2 结合，既可以特征选择，又能处理强共线性。
    * 补充：**随机森林**（Random Forest）
        * 随机森林是一种**集成学习**（ensemble learning，“三个臭皮匠，顶个诸葛亮”）算法，通过**构建多棵决策树并让它们共同投票**，得到更稳定、更准确的预测结果。
        * 核心思想：
            * Bootstrap 抽样：从原始数据中有放回地随机抽取多组数据集
            * 随机特征选择：每棵树训练时只随机选用部分特征
            * 多树投票：
                * 分类：多数树的结果作为最终类别
                * 回归：多棵树结果取平均
        * 优点是不容易过拟合，能处理非线性、交互效应，可给出特征重要性，对异常值、缺失值相对鲁棒；
        * 缺点是可解释性不如逻辑回归、Cox，训练高维数据时速度较慢，无法像回归模型直接给出 OR/HR。
    * 补充：**XGBoost**
        * XGBoost（Extreme Gradient Boosting，极端梯度提升）是梯度提升树（GBDT）的优化升级版，属于集成学习中的 Boosting 算法。
        * 核心思想：
            * Boosting 核心逻辑：串行训练多棵决策树，每一棵新树都专门纠正前一棵树的残差（预测误差），最终将所有树的预测结果加权求和，得到最终输出。
            * 核心优化（具体的实现，不重要！）：二阶泰勒展开损失函数，提升拟合精度；加入正则化项（L1/L2 惩罚），有效防止过拟合；支持并行计算、缺失值自动处理、剪枝优化，训练效率和稳定性大幅提。
        * 优点：
            * 预测精度高，对非线性、交互效应的拟合能力极强；
            * 内置正则化，过拟合风险远低于普通决策树；
            * 支持特征重要性评估，可解释性优于随机森林；
            * 对数据分布要求低，兼容高维、含缺失值的临床数据；
            * 训练速度快，支持并行计算。
        * 缺点：
            * 超参数较多，调参复杂度高于传统统计模型；
            * 可解释性仍弱于 Logistic 回归、Cox 回归等线性模型
            * 对极端异常值敏感，需提前做数据预处理。

        ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260408233131227.png)
