## 模型选择 - 概览
医学中的模型选择主要要看你的结局变量是什么类型，其中下面的1 连续变量用得比较少，2、3比较多。
1. Outcome 为连续变量
    * 线性回归模型 – 正态分布
    * 广义线性回归模型 – 其他分布
2. Outcome 为分类变量
    * **二分类 – Logistic 回归**
        * 是传统方法中用得最多的一类。
    * 多分类 – 变种 Logistic 回归
        * 不建议，一般是把多分类合并成二分类
3. Outcome 为时间 - 事件变量（Time to Event）
    * **无竞争风险 – Cox 回归**
    * 有竞争风险 – Fine-Gray 模型
* 机器学习 vs 传统模型

## 线性回归模型
* 临床建模适用条件 (弱于 LINE 假定)
    * LINE假定是经典线性回归的严格假设，Linear（线性关系）、Independent（独立性）、Normal（正态分布）、Equal variance（等方差）。在临床上，是将LINE的四个严格标准转换成了下列三个弱标准：
    1. 因变量对称分布
    2. 预测因子没有强相关性
    3. 残差无明显趋势
* 举了一个身高预测的例子
    * lm_model = lm(Y ~ H_father + H_mother + Fee)
    * $Y = \alpha + \beta_1 X_1 + \beta_2 X_2 + \beta_3 X_3 + \varepsilon$
        * $\varepsilon$是残差，基本上看到这个残差出现就知道用的是最小二乘法

## 广义线性模型
* 多元线性回归模型（通用形式）：$Y = \alpha + \beta_1 X_1 + \beta_2 X_2 + \beta_i X_i + \varepsilon$
* 对数变换后的线性回归模型：$\log(Y) = \alpha + \beta_1 X_1 + \beta_2 X_2 + \beta_i X_i + \varepsilon$
* 举了一个因变量明显不对称（因而不满足线性回归模型的要求，就不能用线性回归）的例子：
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260407233401435.png)
    * 此时，对Y取对数，它的分布就对称了：
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260407233453185.png)
    * 此时，把取完对数的Y，即$\text{log}(Y)$视为因变量，就可以用线性回归模型了（因此，这里是“广义”线性模型）

## Logistic回归
* Logistic回归的“Logistic”是一个自造词，指的其实就是$\ln\left( \frac{P(Y=1)}{1-P(Y=1)} \right)$
* Logistic 回归的对数似然比（logit）形式
$$\ln\left( \frac{P(Y=1)}{1-P(Y=1)} \right) = \alpha + \beta_1 X_1 + \beta_2 X_2 + \beta_i X_i$$
    * logit 变换形式：对事件发生的 odds（优势比，P/(1−P)）取自然对数，建立与自变量的线性关系，是模型估计的核心形式。
    * 注意，这里没有$\varepsilon$，说明不是通过最小二乘法优化。（实际上是用的极大似然法估计的）

* Logistic 回归的概率形式（Sigmoid 函数）
$$P(Y = 1) = \frac{\exp(\alpha+\beta_1 X_1+\beta_2 X_2+\beta_i X_i)}{1+\exp(\alpha+\beta_1 X_1+\beta_2 X_2+\beta_i X_i)}, \quad
P(Y = 0) = \frac{1}{1+\exp(\alpha+\beta_1 X_1+\beta_2 X_2+\beta_i X_i)}$$
    * 概率形式：通过 Sigmoid 函数将线性预测值映射到[0,1]区间，得到事件发生概率P(Y=1)，同时给出未发生事件的概率P(Y=0)

* 符号说明：
    * $Y=1$：事件发生（阳性结局），$Y=0$：事件未发生（阴性结局）
    * $P(Y=1)$：事件发生的概率
    * $α$：截距项
    * $\beta_1,\beta_2,...,\beta_i$：各自变量的回归系数（对应 OR 值的对数）
    * $X_1,X_2,...,X_i$：预测因子（自变量）
    * $\exp(⋅)$：指数函数，用于将 logit 值转换回概率 / 优势比

* 举了一个抽烟和肺癌的例子（这个例子在视频的10:00，如果没有听懂可以回看）：
    ![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260407234348457.png)
    $$OR = \frac{a/b}{c/d} = \frac{\exp\left( \ln\left( \frac{P(Y=1)}{1-P(Y=1)} \right) \bigg| X=1 \right)}{\exp\left( \ln\left( \frac{P(Y=1)}{1-P(Y=1)} \right) \bigg| X=0 \right)} = \frac{\exp(\alpha+\beta)}{\exp(\alpha)} = \exp(\beta)$$
    * 这是Logistic 回归中优势比（Odds Ratio, OR）的推导公式。
    * （SXS）这里的计算方式对你来说应该不是很重要，但是它能解释为什么之前第三章中，需要对estimate取一个指数才能得到OR值。这里的数学推导看一看就行，重要的是你会用，而不是具体的数学原理。

## Cox回归
* UP主说，Cox回归也是类似于广义线性回归（不过更广义一些，形式和之前的线性回归差得更远）。
* 在医学中，**依然不关心它的风险值，而是关心它的风险比**，即每个自变量变动一个单位，风险单位会跟着变动多少。在医学中，常报告它的HR值（下面第5条）。
* 核心模型公式：
    $$h(t,X) = h_0(t)exp(\beta_1X_1 + \beta_2X_2 + \beta_iX_i)$$
1. 生存函数：具有变量 $X=(X_1, X_2, ..., X_m)$ 的观察对象的生存时间 $T$ 大于某时刻 $t$ 的概率，
    $$S(t,X) = P(T > t, X)$$
    称为生存函数。生存函数 $S(t,X)$ 又称为累积生存率。
2. 死亡函数：具有变量 $X$ 的观察对象的生存时间 $T$ 不大于某时刻 $t$ 的概率，
    $$F(t,X) = P(T \leq t, X)$$
    称为死亡函数。死亡函数 $F(t,X)$ 的实际意义是当观察随访到 $t$ 时刻的累积死亡率。
3. 死亡密度函数：具有变量 $X$ 的观察对象在某时刻 $t$ 的瞬时死亡率，称为死亡密度函数。
    $$f(t,X) = \lim_{\Delta t \to 0} \frac{P(t < T \leq t+\Delta t, X)}{\Delta t} = F'(t,X)$$
4. 危险率 (风险) 函数：具有变量 $X$，且生存时间已达到 $t$ 的观察对象在时刻 $t$ 的瞬时死亡率，
    $$h(t,X) = \lim_{\Delta t \to 0} \frac{P(t < T \leq t+\Delta t | T \geq t, X)}{\Delta t} = \frac{f(t,X)}{S(t,X)}$$
    称为危险率函数。危险率函数 $h(t,X)$ 实际上是一个条件瞬间死亡率。
5. 风险比（hazard ratio, HR）：
    $$HR = \frac{h(t,X = 1)}{h(t,X = 0)} = \frac{\exp(\beta)}{\exp(0)} = \exp(\beta)$$

## 机器学习
### 模型概览
* 正则化技术 (岭回归、LASSO 回归、弹性网络)
    * 最常见。这三者都是对自变量系数进行一定惩罚（LASSO回归可以把自变量系数归到0）。
    * LASSO回归非常“神奇”，可以套在许多线性回归方法之后，只是在现有的方法上进行优化。听UP主的意思，LASSO回归更像是一种提高模型泛化性的技术。
* 支持向量机
* 朴素贝叶斯
* 集成方法 (随机森林、XGBoost)
* 神经网络
    * 上述四种都是传统的机器学习方法，神经网络是机器学习下的一个分支，深度学习。

### 优缺点
* 优点
    1. 对数据的要求灵活
        * 对分布要求低
        * 输入维度高
        * 可输入图像
    2. 模型精度更高
        * 自动学习特征
        * 非线性关系

* 缺点
    1. 稳定性差
        * 随机性
        * 容易过拟合
    2. 速度慢
        * 其实不是大问题；现在的GPU训练模型其实非常快（除了医学影像类的，其实做个分类做个预测训练起来没多慢）；
        * 还提到了一个随机种子的问题，这个确实有一定影响，但是哪怕在深度学习领域，大家做个3-5个也差不多了；
    3. 可解释性低
