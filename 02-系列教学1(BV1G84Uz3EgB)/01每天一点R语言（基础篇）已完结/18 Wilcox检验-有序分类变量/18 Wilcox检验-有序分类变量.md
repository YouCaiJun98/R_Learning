* 本节课其实是对上节课的补充，上节课只教了两种面向数值向量的wilcox检验，这节课补充了对**分类变量**的wilcox检验。
* 本节课举的例子是，对于A/B两个分组，想看看哪个分组的肿瘤分期（Stage向量）更“高”；
* Q：直接用符号去做检验肯定是有问题的，因为检验只能对两个数值向量做，那该怎么办呢？
* A：将符号（比如这里的“肿瘤分期”）**解释**为数值！
    * 在构造factor时，指定`ordered = T`，即认为不同符号之间是有**先后顺序**的，这样它们就隐含了数值！比如，对于肿瘤分期，"I > II > III > IV"（你可以理解成，在后台，将标签"I"对应了数值"1"，不同的标签对应了不同的数字，就可以按数字将这几个标签排序了）；
    * 对于有顺序的符号，将它们进一步视为数值（即下文提到的“显式转换”），然后就可以使用wilcox检验了！
* 提到了一个“显式转换”，`as.numeric`：
    ```R
    as.numeric(<向量>)
    ```
    * 它的作用是将非数值向量（如这个例子中的`factor`，它也是一种特殊的数值格式）转换成数值形式；
    * 它的本质是**强制类型转换**（其他语言中的称呼），例如，将一个浮点（带小数点的数）转换成整形（整数）：
        ```C++
        int a;
        float b = 1.5;
        a = static_cast<int>b; # a = 1
        ```
    * 当然，`R`里还有很多其他的显示转换，比如将向量转换成字符向量：
        ```R
        as.character()
        ```
