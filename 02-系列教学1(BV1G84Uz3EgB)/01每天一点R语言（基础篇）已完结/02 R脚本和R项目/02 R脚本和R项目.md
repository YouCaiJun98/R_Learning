## 创建R脚本
* （视频 0:24）创建`R`脚本：在`R Studio`内，点击左上角`File -> New File -> R Script`，即可在左侧控制区新建一个`R`脚本（布局看着挺像Matlab的）。
![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/8de9dc84c261de3017dab6268886d3e4.png)
* 在上图中，还能看到其他不同语言的脚本，例如C/C++文件、Markdown文件（这个.md文件就是markdown语法写出来的文件）、HTML文件（网页文件）、Python文件等等；
* （视频 1:37）Q：为什么要在`R`脚本中写代码，而不是在左下角的控制台（console）里写？
    * A：大概有这么几种原因：①为了能复用写好的代码；②console里写的代码无法保存；③方便编写大段落的代码；

## 创建R项目
* （视频 2:20）在`R Studio`中，点击右上角`Project: (None) -> New Project...`可以创建新`R`项目
![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260322231748001.png)
* Q：什么是项目（proj）？为什么要创建项目呢？
    * A：proj是若干`R`脚本+数据组合成的一套系统，它们搭配起来可以完成一定的功能。当你一个`R`脚本搞不定某个事情时，就以proj的形式组织文件，实现复杂功能。
* 点击创建项目时，会弹出来这个窗口，你需要填的内容包括项目的名称（Directory name）、项目的地址（Create project as subdirectory of），底下的两个（Create a git repository、Use renv with this project）你暂时不用管。
![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260322232115337.png)
    * 当然，我也能推测出来这俩是干啥的。首先，第一个，“Create a git repository”是指用`git`这个项目管理工具来管理你的`R`项目。`git`是负责版本控制、回溯的（例如，你把一段代码改坏了，想回滚到之前的版本，就可以用`git`），是程序员非常常用的工具；“Use renv with this project”应该是使用`R`的虚拟环境，这个和python的逻辑应该很像，“虚拟环境”是指，用软件的手段把不同版本的包隔离开，避免发生版本冲突（“包”你可以理解成`R`的不同功能的插件）。
* 项目创建完后，会在你指定的目录下生成你指定的项目文件，文件后缀是`*.Rproj`，下图中，上方的红框是你的工作目录（你指定的那个文件夹），下方的红框是你生成的项目文件：
![](https://raw.githubusercontent.com/YouCaiJun98/MyPicBed/main/imgs/20260322232833870.png)
