## 本课概览
* 目标：搭建多因素模型，能讲清楚“调整后 OR”的含义
* 核心内容：
    * 建立多因素模型：glm(y ~ x1 + x2 + x3, family=binomial())
    * 共线性
* 输出：多因素OR表
* Q：为什么要进行多因素回归？
    * A：在协变量相同的前提下，增加一个分析的因变量，以排除干扰，确定独立因素。

| 对比维度 | **单因素 Logistic** | **多因素 Logistic** | 为什么要做多因素？ |
| :--- | :--- | :--- | :--- |
| **视角** | 只看一个变量 | 同时看多个变量 | **更全面、更客观** |
| **结果** | **粗OR** (Crude OR) | **调整后OR** (Adjusted OR) | **粗OR可能是假的，调整后才可信** |
| **目的** | **筛选**（找出有潜力的因素） | **推断/结论**（排除干扰，确定独立因素） | **这是论文入编的硬性要求** |
| **适用场景** | 数据探索、初步筛选 | 最终结论、临床指导意义 | **必须做，否则结论站不住脚** |

* UP主提到医学中常用的思路包括：
    * 先看单因素的结果，将$P$值小于一定门槛的变量作为多因素分析的基础。（UP主说，这个思路在医学上很多人使用，但是在统计学上是有问题的）
    * 更好的思路是，看临床（clinical-oriented model，即有没有临床意义）。如果是一个new marker（*啥意思，一个新的分析变量吗*），可以用看$P$值的方法；对于公认显著的因素，哪怕$P$值并不显著，也应将它们纳入最终的模型里。

## 多因素建模
* 相比单因素建模，多因素建模的代码非常好修改，只要修改`glm()`中的第一项，`y ~ x`即可，将它修改成`y ~ x1 + x2 + ...`，例如课程中的例子：
    ```R
    model_multi <- glm(TreatmentOutcome ~ AGE_AT_DIAGNOSIS +
                        ER_STATUS + 
                        HER2_STATUS + 
                        TUMOR_STAGE, family = binomial(), data = data_analysis)
    summary(model_multi)
    ```
* 剩下的步骤就和单因素建模里的一样啦。
