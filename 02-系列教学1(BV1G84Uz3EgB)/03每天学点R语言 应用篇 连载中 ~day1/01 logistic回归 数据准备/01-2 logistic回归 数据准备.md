## Tidyverse版本数据分析
* 豆知识：`Rmd`文件默认的路径是和这个文件同级的目录，和`R proj`不在同一个路径下。
* UP主介绍了一个新的函数，`everything()`。这个函数是`tidyselect/dplyr`里的列选择助手函数，核心作用是选中 数据框/tibble 的所有列，最常用于`select()`、`across()`等函数中，实现列重排、批量操作等。
* 又用到了另一个函数，`across()`。这个函数是批量对多列执行同一个函数（说起来有点绕，但你可以理解为，对选中的每一列，都按照传入的那个函数进行处理），基本语法如下；
    ```R
    across(.cols, .fns, ..., .names = NULL)
    ```
    其中，
    * `.cols`：表示选哪些列（列名、数值列、匹配规则、`everything()`都能写）
    * `.fns`：表示要执行的函数（1个 / 多个，内置 / 自定义都行）
    * `.names`：自定义新列名格式
    这个函数有以下典型的**应用场景**：
    * summarise + across → **批量多列统计**，例如对所有数值指标算均值、标准差
        ```R
        # 模拟临床数据
        df <- tibble(
        分组 = c("对照","治疗"),
        年龄 = c(55,60),
        血糖 = c(5.2,7.8),
        血压 = c(120,135)
        )

        # 所有数值列，批量算均值
        df %>%
        group_by(分组) %>%
        summarise(
            across(where(is.numeric), mean, na.rm = TRUE)
        )
        ```
    * mutate + across → 批量多列改数据，例如对所有数值列标准化 / 取整 / 替换缺失
        ```R
        # 所有数值列四舍五入保留1位
        df %>%
        mutate(
            across(where(is.numeric), ~round(.x, 1))
        )
        ```
    * 多函数并行 + 自定义列名，例如一列同时算均值 + 方差，自动命名
        ```R
        df %>%
        summarise(
            across(
            c(年龄,血糖),
            list(均值=mean, 标准差=sd),
            na.rm=TRUE,
            .names = "{.col}_{.fn}"
            )
        )
        ```
* UP主写了一句很抽象的代码，我们来逐层解析：
    ```R
    data_analysis %>% summarise(across(everything(), ~ sum(is.na(.))))
    ```
    * 首先，管道符和summarise我们已经很熟悉了，就跳过了；
    * 从`summarise()`最内层看起，最内层是`is.na(.)`，这里的`.`是占位符，语义是**当前处理的这一列**；
    * 往外跳出一层，`sum(is.na(.))`的意思是，`is.na(.)`返回一个bool向量，因为这个向量的内容本质上是0和1（0代表`FALSE`，1代表`TRUE`），所以可以求和；
    * 再往外跳出一层，`~sum(is.na(.))`中，`~`的语义是，**对每一XX，都进行某操作**。在这个函数里，它的意思就是，对于每一列，都进行求和操作；
    * **`~` + `.`是一对常用的固定搭配**！它们是`tidyverse`**匿名函数**的标准简写格式！语义是，"~ 对 . 做什么操作"。例如，
        ```R
        df %>% mutate(新列 = 原列 * 2)
        # 等效于
        df %>% mutate(新列 = ~ . * 2)
        ```
        它的语义是，对于每一个（"~"）原列（"."），都做原列数值乘2的操作！
        * 那么，什么是**匿名函数**呢？一般来说，我们在定义一个函数的时候，会给这个函数起个名字，例如，
        ```R
        # 给函数起名：count_na
        count_na <- function(x) {
            sum(is.na(x))
        }
        # 要用的时候：喊名字调用
        count_na(iris$Sepal.Length)
        ```
        * **匿名函数**就是，用完就扔、临时用一下、不用起名的函数！有点类似`python`中的lambda语法！例如，
        ```R
        # 完整版匿名函数
        function(x){ sum(is.na(x)) }

        # 简写版匿名函数（~ + .）
        ~ sum(is.na(.))
        ```
    * 再往外跳一层，`across(everything(), ~ sum(is.na(.)))`的意思是，先选中了所有列（通过`everything()`实现的，它也是`across()`这个函数第一个传入的参数），然后对每一列，都进行`~ sum(is.na(.))`。
* UP主又介绍了一个函数，`pivot_longer()`，它的作用是，把“很宽”的表格，拉长变成“长表格”。这里，“宽表”是指很多列（比如：血压 1、血压 2、血压 3、血糖 1、血糖 2…）的表，而“长表”是指只有两列的表，内容是“变量名 + 对应数值”。
