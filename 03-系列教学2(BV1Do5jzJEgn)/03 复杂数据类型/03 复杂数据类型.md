## 矩阵
* 将数据按行与列组织，相当于二维数组。
* 和向量类似，每个数据也要求是同样的数据类型。
* 基本的语法如下：
    ```R
    matrix(数据, nrow=行数, ncol=列数, byrow=是否按行填充)
    ```
    其中，
    * nrow：设置一共有几行
    * ncol：设置一共有几列
    * byrow=FALSE（默认）：按列填
    * byrow=TRUE（默认）：按行填
* 访问矩阵元素时，应该按照这个规则进行访问：`m[行号, 列号]`，方括号内第一个是行数，第二个是列数。某个空着不写，比如`m[行号, ]`就是访问这一整行的元素。
    * 此外，还可以用逻辑型向量、行名&列名来索引！
* 在修改矩阵某一行/某一列的时候，新的赋值向量一定要和原行/列长度一致。如果用一个标量赋值，会将整行/整列都修改成这个值。
* 使用`t()`函数可以进行矩阵的转置。
* 和数据框的不同：数据框里的元素类型可以不同，且数据框不能转置。


## 数组
* 数组的数据类型也必须是统一的。
* `array()` 是 `R` 中创建**多维数组**的函数，是矩阵（matrix，二维特例）的**通用形式**，用于存储任意维度、元素类型统一的数据。
* 基本的语法如下：
    ```R
    array(数据, dim = c(维度1, 维度2, ..., 维度n))
    ```
    其中，
    * 数据：填充数组的元素（需类型统一，如全数值、全字符）
    * dim：指定数组的维度，例如 `c(2,3,4)` 表示 2×3×4 的三维数组
    * 若省略 dim，会生成与输入数据长度相同的一维向量（**等价于普通向量**）

## 数据框
* `R`中最重要的数据类型。
* 和矩阵一样，也是二维，但是数据类型可以不一样。
* 一般每列叫做一个变量，每行叫做一个观测样本。
* 和课程不同的是，4.5版本的`R`在创建数据框时，不会自动把字符型的数据转换成因子型的。
* 访问数据时，不同的访问方式会得到不一样的结果，例如，
    ```R
    student['ID']
    ```
    的返回结果是一个数据框；
    ```R
    student[, 'ID']
    ```
    的返回结果是一个向量。这个访问方式等价于`student$ID`。

## 列表
* 列表是`R`中最复杂的数据类型，是数据对象的有序的集合，可以存储**任意类型**、**任意长度**、**任意结构**的数据。
    * 列表中元素的数据类型可以各不相同，而且可以是不同的数据结构（可以是向量，矩阵等等，甚至也可以是另一个列表！）。
    * 列表中每个元素的长度也可以各不相同！
* 基本的语法如下：
    ```R
    list(元素1, 元素2, 元素3, ..., 名称1 = 元素A, 名称2 = 元素B, ...)
    ```
    元素可以是：向量、矩阵、数据框、函数、甚至另一个列表（支持嵌套）
* 访问列表元素需要用双重方括号（`[[]]`），否则（用单方括号`[]`）将会返回一个列表。
